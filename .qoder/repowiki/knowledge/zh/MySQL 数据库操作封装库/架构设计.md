- **核心入口**：`SQLExecutor` (`lazy_mysql/executor.py`) 作为统一门面，管理数据库连接生命周期（含自动重连与事务提交）并分发 CRUD 请求。
- **分层设计**：采用“执行器-工具函数”分离架构。`executor.py` 负责连接管理与错误重试；具体 SQL 逻辑委托给 `utils/` 下的模块化函数（如 `select.py`, `insert.py`）。
- **配置与模型**：使用 Pydantic 定义 `MySQLConfig` 和 `FetchConfig`，实现配置解析（支持环境变量注入）与查询结果格式化的强类型约束。
- **辅助工具**：`tools/` 包提供 `build_where_clause` 等 SQL 片段生成器，支持动态条件构造与防注入处理。
