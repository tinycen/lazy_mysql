### 问题 5：`add_limit()` 存在 SQL 注入风险（字符串插值）

**文件**：`lazy_mysql/tools/sql_utils.py`
**行号**：第 38-42 行
**级别**：🟠 注意问题

**问题描述**：
`add_limit()` 函数通过 f-string 直接将整型和浮点型值插入 SQL 字符串，而非使用参数化查询。虽然整型/浮点型注入风险相对较低，但这是与库其他部分（如 `execute()`、`build_where_clause()`）参数化风格不一致的危险先例——一旦此函数被扩展以支持更复杂的输入格式，极易引入 SQL 注入漏洞。

**代码片段**：
```python
if isinstance(value, (int, float)):
    condition = f"{column_alias}{column} {operator.upper()} {value}"  # 直接插值
else:
    condition = f"{column_alias}{column} {operator.upper()} '{value}'"  # 字符串插值更危险
```

**修复建议**：
改为参数化风格，使函数返回 `(condition_str, params_list)` 元组，由调用方拼接进 SQL：

```python
def add_limit(column, value, column_alias="", add_and=True, operator="="):
    if value in ("", "all", "null", None, [], ()):
        return "", []

    if column_alias:
        col = f"{column_alias}.{column}"
    else:
        col = column

    if operator.upper() in ('IN', 'NOT IN') and isinstance(value, (list, tuple)):
        placeholders = ', '.join(['%s'] * len(value))
        condition = f"{col} {operator.upper()} ({placeholders})"
        params = list(value)
    elif isinstance(value, (int, float)):
        condition = f"{col} {operator.upper()} %s"
        params = [value]
    else:
        condition = f"{col} {operator.upper()} %s"
        params = [value]

    prefix = "AND " if add_and else ""
    return f"{prefix}{condition}", params
```
