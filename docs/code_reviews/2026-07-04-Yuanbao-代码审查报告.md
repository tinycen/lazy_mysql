### 问题 1：SQL注入风险——多处表名/视图名直接拼接到SQL

**文件**：`lazy_mysql/executor.py`、`lazy_mysql/utils/insert.py`、`lazy_mysql/tools/table_export.py`
**级别**：🔴 严重bug

**问题描述**：
`table_name` 和 `view_name` 参数在多个方法中被直接字符串拼接到 SQL 语句中，没有任何输入校验。虽然 `table_export.py` 中部分方法使用了正则校验，但 `executor.py` 及 `utils/insert.py` 中的所有方法均未对表名进行过滤，可被恶意表名攻击（如 `users; DROP TABLE users;--`）。

**代码片段**：
```python
# executor.py - select() 等多个方法
sql = f"SELECT {distinct_clause}{select_clause} FROM {table_names}"

# insert.py - _upsert_single(), _upsert_batch()
insert_sql = f"INSERT INTO {table_name} ..."

# table_export.py - _get_view_source()（无任何校验）

```

**修复建议**：
1. 在所有使用表名的地方统一添加正则白名单校验：
   ```python
   import re
   _TABLE_NAME_PATTERN = re.compile(r'^[a-zA-Z_][a-zA-Z0-9_]*$')
   def _validate_table_name(name: str) -> str:
       if not _TABLE_NAME_PATTERN.match(name):
           raise ValueError(f"Invalid table name: {name}")

   ```
2. `table_export.py` 中 `_validate_table_name()` 函数已在多处使用，应将其移至公共模块并复用

### 问题 2：空列表params导致SQL参数丢失

**文件**：`lazy_mysql/executor.py`
**级别**：🔴 严重bug

**问题描述**：
当 `params=[]`（空列表）时，判断 `if params:` 为 `False`，导致调用 `self.mycursor.execute(sql)` 时**不传入参数**。这使得占位符 `%s` 失去实际参数绑定，可能导致数据库报错或查询结果异常，且难以定位原因。

**代码片段**：
```python
def execute(self, sql, params=None, ...):
    if params:  # [] 在此为 False
        # ...
    else:
        self.mycursor.execute(sql)  # params=[] 时走此分支，参数丢失！
```

**修复建议**：
```python
if params is not None:
    self.mycursor.execute(sql, params)
else:
    self.mycursor.execute(sql)
```

### 问题 3：`self_close` 参数在 executemany 批量插入时被忽略

**文件**：`lazy_mysql/utils/insert.py`

**问题描述**：

**代码片段**：
```python
def _executemany_optimized(executor, table_name, fields, skip_duplicate=False,
                          commit=True, batch_size=10000, self_close=False):

```

**修复建议**：
```python
executor.execute(sql, values, commit=commit, self_close=self_close)
```

### 问题 5：类型注解使用过时语法（Python 3.10+ 项目）

**文件**：`lazy_mysql/models/fetch_config.py`
**级别**：🟠 注意问题

**问题描述**：
使用 `typing.List` 和 `Optional[List[str]]` 等 Python 3.9 以下语法。项目的 `setup.py` 要求 `python_requires='>=3.10'`，应使用内置类型注解和 `X | None` 语法。ruff 检测到 3 处过时语法（UP045、UP006、UP035）。

**修复建议**：
```python
# 当前
from typing import List, Literal, Optional
data_label: Optional[List[str]] = Field(default=None, ...)

# 建议
from typing import Literal
data_label: list[str] | None = Field(default=None, ...)
```
