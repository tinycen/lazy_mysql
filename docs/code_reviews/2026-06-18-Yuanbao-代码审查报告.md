### 问题 1：SQL 注入漏洞 — `add_limit()` 直接拼接用户输入

**文件**：`lazy_mysql/tools/sql_utils.py`
**行号**：15-25
**级别**：🔴 严重bug

**问题描述**：
`add_limit()` 函数将用户传入的 `value` 直接字符串插值到 SQL 语句中，完全绕过了参数化查询机制。虽然该函数设计为内部工具函数，但当调用方传入不可信的用户输入时（如 Web 请求参数），将造成 SQL 注入漏洞。

**代码片段**：
```python
def add_limit(column, value, column_alias="", add_and=True, operator="="):
    # ...
    if isinstance(value, (int, float)):
        condition = f"{column_alias}{column} {operator.upper()} {value}"
    else:
        condition = f"{column_alias}{column} {operator.upper()} '{value}'"   # 直接拼接！
```

**修复建议**：
重构为参数化查询，使用 `%s` 占位符，返回 `(sql_fragment, params)` 元组：

```python
def add_limit(column, value, column_alias="", add_and=True, operator="="):
    if value in ("", "all", "null", None, [], ()):
        return ""

    if column_alias != "":
        column_alias += "."

    if operator.upper() in ('IN', 'NOT IN') and isinstance(value, (list, tuple)):
        placeholders = ', '.join(['%s'] * len(value))
        condition = f"{column_alias}{column} {operator.upper()} ({placeholders})"
        params = value
    elif isinstance(value, (int, float)):
        condition = f"{column_alias}{column} {operator.upper()} %s"
        params = (value,)
    else:
        condition = f"{column_alias}{column} {operator.upper()} %s"
        params = (value,)

    prefix = "AND " if add_and else ""
    return prefix + condition, params
```

> ⚠️ **注意**：此修改为破坏性变更，所有调用方需同步适配。

### 问题 2：表名/库名直接字符串插值 — `table_export.py` 多处 SQL 注入风险

**文件**：`lazy_mysql/tools/table_export.py`
**行号**：17、25、31、37
**级别**：🔴 严重bug

**问题描述**：
`table_md()` 函数中多处将 `table_name` 直接插入 SQL 字符串（`SHOW TABLES LIKE '{table_name}'`、`SHOW FULL COLUMNS FROM {table_name}` 等），如果 `table_name` 由用户传入且未做严格校验，将产生 SQL 注入漏洞。

**代码片段**：
```python

# 第 25 行
query = f"SHOW FULL COLUMNS FROM {table_name}"
# 第 31 行
executor.execute(f"SHOW KEYS FROM {table_name} WHERE Key_name = 'PRIMARY'", self_close=False)
# 第 37 行
executor.execute(f"SHOW INDEX FROM {table_name}", self_close=False)
```

**修复建议**：
对 `table_name` 进行严格白名单校验（仅允许字母、数字、下划线，且非空）：

```python
import re

def _validate_table_name(table_name: str) -> str:
    if not re.match(r'^[a-zA-Z_][a-zA-Z0-9_]*$', table_name):
        raise ValueError(f"Invalid table name: {table_name!r}. Table names must be alphanumeric identifiers.")
    return table_name
```

在 `table_md()` 入口处调用：
```python
table_name = _validate_table_name(table_name)
```


在 `delete.py` 中增加与 `update.py` 等效的空值检查：

```python
def delete(executor, table_name, conditions, commit=False, self_close=False):
    if not conditions:
        if self_close:
            executor.close()
```


**修复建议**：
